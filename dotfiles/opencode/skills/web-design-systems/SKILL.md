## What I do
- Translate product goals into a concrete visual and interaction system.
- Produce responsive UI with strong typography, clear spacing rhythm, and visual hierarchy.
- Prioritize accessibility: semantic structure, keyboard support, contrast, and reduced motion fallbacks.
- Balance polish with maintainability by using reusable components and predictable CSS architecture.

## Working style
- Start with layout and information architecture before visual details.
- Define constraints first: breakpoints, spacing scale, type scale, color tokens, motion rules.
- Prefer meaningful animation for transitions and state changes, avoid decorative noise.
- Validate states: loading, empty, error, success, and edge-content overflow.

## Output checklist
- Mobile and desktop both verified.
- Components are reusable and named clearly.
- No regressions to existing design system conventions.
- A brief rationale explains visual choices and tradeoffs.
